# Тетрис — руководство пользователя

Веб-сервис квартального планирования портфеля задач (аналог Excel-«Тетриса»).
Данные о задачах приходят из Jira; планирование, оценки и оформление — здесь.

---

## 1. Вход и роли

1. Откройте сайт, введите почту и пароль.
2. Тестовые пользователи: `editor@askona.ru` (редактор), `reader@askona.ru` (читатель).
3. Роль видна справа сверху:
   - **Редактор** — может всё: создавать метаспринты/планы, добавлять задачи, менять оценки, оформлять.
   - **Читатель** — только просмотр (🔒), ничего не меняет.

---

## 2. Из чего всё состоит

- **Метаспринт** — период планирования (квартал/месяц). Внутри него — планы.
- **План** — конкретная итерация планирования (версия). В метаспринте их может быть несколько.
- **Автовыгрузка** — общий пул ВСЕХ задач из Jira, сгруппированных по «колодцам». Отсюда задачи добавляются в план.
- **Таблица плана** — задачи, которые вы отобрали в текущий план.

Внизу экрана две строки-переключателя:
- **ряд 1:** кнопка «📥 Автовыгрузка» и список **метаспринтов**;
- **ряд 2:** список **планов** выбранного метаспринта и их статусы.

---

## 3. Метаспринты

- **Создать:** внизу в ряду метаспринтов — кнопка **«+ метаспринт»**. Введите название.
- **Выбрать:** клик по названию метаспринта.
- **Переименовать:** **двойной клик** по названию метаспринта → впишите новое → Enter.
- **Удалить:** кнопка **«🗑 удалить метаспринт»** (удалит метаспринт со всеми его планами, необратимо).

---

## 4. Планы

В ряду «Планы:» (для выбранного метаспринта):

- **Создать пустой план:** кнопка **«+ план»**.
- **Создать на основе текущего (форк):** кнопка **«⎘ создать на основе»** — новый план копирует ВСЁ из текущего (задачи, раскладку, заметки, остатки SP). Удобно для итераций: зафиксировали план 1 → форкнули в план 2 → правите дальше, а старый остаётся для сравнения.
- **Выбрать план:** клик по названию.
- **Переименовать:** **двойной клик** по названию плана.
- **Удалить:** кнопка **«🗑 удалить»**.
- **Статус плана** (выпадающий список «статус:»):
  - **черновик** — рабочий, всё редактируется;
  - **утверждён** — план «заморожен»: данные сохраняются снимком на момент утверждения и больше не меняются от обновлений Jira (историчность). Редактирование заблокировано;
  - **архив** — тоже заморожен (снимок), не редактируется.
  - Вернуть «утверждён»/«архив» обратно в **черновик** → план снова станет живым и редактируемым.
  - У замороженного плана в шапке таблицы виден бейдж **«🔒 заморожен (снимок)»**.

---

## 5. Автовыгрузка — отбор задач в план

Вкладка **«📥 Автовыгрузка»** (внизу слева). Здесь все задачи, разложенные по колодцам:
**MetaSprint, AlphaSprint, OpenSprint, To be allocated.**

Как задача попадает в колодец (автоматически):
- есть метка `MetaSprint…` → **MetaSprint**; метка `AlphaSprint` → **AlphaSprint**;
- если меток нет, но у задачи есть **активный спринт** (в Jira state=active) → **OpenSprint**;
- остальное → **To be allocated**.

Действия:
1. Сначала **выберите метаспринт и план снизу** (куда добавлять).
2. Найдите задачи: поиск по Key/Summary сверху, сортировка (клик по заголовку), фильтры (см. п.7–8).
3. Нажмите **«+»** слева у задачи — она добавится в выбранный план. «✓» — уже в плане.
4. Двойной клик по строке — открыть карточку задачи.

---

## 6. Таблица плана

Переключитесь на нужный план (ряд «Планы»). Здесь — только задачи этого плана.

- **Порядок строк:** тяните за маркер **≡** слева (ручной порядок, общий для всех — хранится в БД). Если включена сортировка по столбцу, нажмите «↕ ручной порядок», чтобы вернуть ручной режим и снова таскать.
- **Сортировка:** клик по заголовку столбца (повторный — обратный порядок).
- **Ширина столбца:** тяните правую границу заголовка.
- **Порядок столбцов:** тяните заголовок за значок **⠿** (порядок личный, хранится в браузере).
- **Выделить строку:** клик по задаче — подсветится синей рамкой (удобно не терять её при горизонтальной прокрутке). Повторный клик снимает.
- **Убрать задачу из плана:** кнопка «−» у Key.
- **ИТОГО по плану** и **Загрузка платформ** — строки под таблицей (см. п.10–11).

---

## 7. Меню «Колонки» (вверху)

Показать/скрыть столбцы, в т.ч. группы:
- **Платформы — оценки (17)** — зелёные столбцы оценок по платформам;
- **Спринты по платформам (17)** — фиолетовые столбцы «за сколько спринтов платформа сожжёт задачу».

Настройка видимости — личная (хранится в браузере).

---

## 8. Фильтр по оценённости («Фильтр ▾», вверху)

Прячет/помечает «неготовые» задачи. 4 независимые категории, у каждой **2 галочки** («показывать задачи» и «значок ⚠») и счётчик количества:
- **Недооценённые** — у выбранной платформы нет оценки;
- **Без платформ и без оценок** — пусто;
- **Есть оценка, но команда не выбрана**;
- **Оценка у невыбранной платформы**.

Наведите на **⚠** у задачи — увидите причину. Настройки личные (браузер).

---

## 9. Фильтр по столбцам (воронки в заголовках)

У каждого заголовка — значок-воронка **▼**. Клик открывает мини-фильтр:
- числовые столбцы / платформы / спринты: **«только с значением»** + диапазон «от … до»;
- текстовые: **«содержит …»**.

Главный сценарий: дозагрузить недозагруженную платформу — открыть воронку у столбца платформы, поставить «только с значением» → останутся только задачи с оценкой по этой платформе. Комбинируется с сортировкой. Фильтры личные (браузер), сбрасываются кнопкой «сбросить».

---

## 10. Промежуточные итоги (кнопка в плане)

Кнопка **«Промежуточные итоги»** в панели плана:
- выберите столбец и функцию (**сумма / среднее / медиана / мода**), нажмите «+ Добавить итог»;
- можно добавить **несколько** итогов — они показываются чипами под таблицей; крестик убирает;
- считаются по данным (работают даже для скрытых столбцов), большие числа с разделением разрядов;
- настройки личные (браузер).

---

## 11. Velocity метаспринта и перегруз платформ

Кнопка **«📊 Velocity метаспринта»** (редактор) — для каждой платформы в текущем метаспринте задаются:
- **Ёмкость за метаспринт (SP)** — сколько платформа осилит; пусто = без лимита;
- **SP за спринт** — делитель для колонок «спринты» (по умолчанию 5, у BPMSoft 6).

Под таблицей плана — строка **«Загрузка платформ»**: по каждой платформе показано `спрос / ёмкость`. Если спрос (сумма оценок плана) превышает ёмкость — платформа подсвечивается **красным** (перегруз). Так видно, где пора остановиться.

---

## 12. Карточка задачи (двойной клик по строке)

Вкладки: **Бизнес-информация, Техническая информация, Приоритизация, Подтверждение результата.**

- Поля редактируются (могут отличаться от Jira).
- На вкладке «Техническая информация»:
  - **МАКСИМУМ спринтов** (0/пусто = считается автоматически);
  - **Оценки по платформам** (вне плана — базовые из Jira);
  - **Остаток SP в этом плане** (когда карточка открыта из плана) — переопределяет оценку Jira только для этого плана; кнопка **«↺ Сбросить остатки к Jira»** возвращает оценку Jira.
- В таблице плана **изменённые остатки выделены цветом** — видно, какие задачи вы уже «потрогали»; после сброса к Jira цвет снимается.
- «Обновить» — сохранить; «⟳ Запись в Jira (этап 12)» — заглушка (реальная запись в Jira появится с вебхуком).

---

## 13. Оформление таблицы (как в Excel)

- **Кисть 🖌** — выберите цвет и закрашивайте ячейки/строки; ластик снимает заливку. Отмена: **Ctrl+Z** (по шагам), **Esc** (всё с момента включения кисти).
- **+ заметка** — строка-заметка между задачами (двойной клик — ввести текст). Общая (хранится в БД).
- **Проверка DoD / Проверка CEO** — подсветка задач по условию (нет DoD / есть приоритет CEO).
- Заливки, заметки, порядок строк — общие для всех (в БД). Ширины, порядок столбцов, палитра, фильтры, итоги — личные (в браузере).

---

## 14. Обновление данных из Jira

- Кнопка **«⟳ из Jira»** (редактор) — перечитать задачи из выгрузки. Повторная загрузка обновляет задачи, не плодит дубли (опознаются по внутреннему id).
- В будущем (этап 12) это станет автоматическим через вебхук — задачи будут подтягиваться и обновляться сами.
- Утверждённые/архивные планы при этом **не меняются** (заморожены снимком).

---

## 15. Типовой сценарий работы (коротко)

1. Создать **метаспринт** (или выбрать существующий).
2. Создать в нём **план** (+ план) — или «⎘ создать на основе» прошлого.
3. Перейти в **Автовыгрузку**, выбрать метаспринт/план снизу, нажимать **«+»** у нужных задач.
4. Перейти в **план**: задать **Velocity метаспринта**, смотреть **загрузку платформ** (перегруз — красным).
5. Открывать карточки, проставлять **остатки SP**, фильтровать/сортировать, оформлять.
6. Свести **промежуточные итоги** (суммы по нужным столбцам).
7. **Утвердить** план (заморозить для истории) и **форкнуть** в следующий — для новой итерации.
